The Temporal Dead Zone (TDZ) is a behavior in JavaScript where a variable declared with let or const cannot be accessed before its declaration is reached in the code.

Definition

The Temporal Dead Zone is the time between:

entering a block scope, and
the point where the variable is declared and initialized.

During this period, accessing the variable causes a ReferenceError.



Example
{
    console.log(a); // ❌ ReferenceError

    let a = 10;

    console.log(a); // ✅ 10
}


Why Error Happens?
The variable a is created when the block starts.
But it stays in the Temporal Dead Zone until the line:
let a = 10;

Before that line executes, JavaScript does not allow access to a.

Another Example with const
console.log(x); // ❌ ReferenceError

const x = 5;

Same reason:
x exists in memory but is unavailable until initialization.

JavaScript Classes

A class in JavaScript is a blueprint for creating objects.

It helps organize code using:

properties (variables)
methods (functions)

JavaScript classes were introduced in ES6 (ECMAScript 2015).

Basic Syntax
class Person {
    constructor(name, age) {
        this.name = name;
        this.age = age;
    }

    greet() {
        console.log("Hello, my name is " + this.name);
    }
}
Creating Object from Class
const p1 = new Person("Rahul", 22);

console.log(p1.name); // Rahul
p1.greet(); // Hello, my name is Rahul





Getter and Setter in JavaScript
Definition
Getter

A getter is a method used to read or access a property value.

Setter

A setter is a method used to update or modify a property value.

They provide controlled access to object properties.

Basic Example
class User {
    constructor(name) {
        this._name = name;
    }

    // Getter
    get name() {
        return this._name;
    }

    // Setter
    set name(value) {
        this._name = value;
    }
}

const user1 = new User("Rahul");

console.log(user1.name); // Getter called

user1.name = "Aman"; // Setter called

console.log(user1.name);
Output
Rahul
Aman

Object Cloning in JavaScript
Definition

Object cloning means creating a copy of an object.

The new object contains the same properties and values as the original object.

Why Do We Need Object Cloning?

Objects are stored by reference in JavaScript.

If we directly assign one object to another variable, both variables point to the same memory location.

Because of this:

👉 Changing one object also changes the other.

To avoid this problem, we use object cloning.

Easy Understanding

Imagine:

👉 Original object = Original notebook

👉 Cloned object = Photocopy of notebook

Now both are separate.

Changes in photocopy do not affect original notebook.




Problem Without Cloning
const user1 = {
  name: "Rahul"
};

const user2 = user1;

user2.name = "Aman";

console.log(user1.name);

Output:

Aman
Why?

Because:

const user2 = user1;

does NOT create a new object.

Both variables point to same object in memory.

Memory Visualization
user1  -------->  { name: "Rahul" }
user2  --------/

Both point to same object.

Method 1 → Object Cloning Using Spread Operator
Definition

Spread operator (...) copies all properties from one object into another object.

Introduced in ES6.

Syntax
const newObject = { ...oldObject };
Example
const user1 = {
  name: "Rahul",
  age: 22
};

const user2 = { ...user1 };

user2.name = "Aman";

console.log(user1);
console.log(user2);

Output:

{ name: "Rahul", age: 22 }

{ name: "Aman", age: 22 }
Explanation

Spread operator creates a new separate object.

Now changing cloned object does not affect original object.

Real-Life Example
const employee = {
  name: "Rohit",
  salary: 50000
};

const updatedEmployee = {
  ...employee,
  salary: 60000
};

console.log(updatedEmployee);

Output:

{ name: "Rohit", salary: 60000 }





------||------




Method 2 → Object Cloning Using Object.assign()
Definition

Object.assign() copies properties from source object into target object.

Syntax
Object.assign(target, source)
Example
const user1 = {
  name: "Rahul",
  age: 22
};

const user2 = Object.assign({}, user1);

user2.name = "Aman";

console.log(user1);
console.log(user2);

Output:

{ name: "Rahul", age: 22 }

{ name: "Aman", age: 22 }
Explanation
{}

creates empty object.

Then properties from user1 are copied into it.

Multiple Object Merge Example
const obj1 = {
  a: 1
};

const obj2 = {
  b: 2
};

const result = Object.assign({}, obj1, obj2);

console.log(result);

Output:

{ a: 1, b: 2 }

Error Handling in JavaScript
Definition

Error handling is the process of detecting and managing errors in a program so that the application does not crash.

JavaScript provides special statements to handle errors safely.

Why Error Handling is Important?

Without error handling:

Program may stop suddenly
Application may crash
Bad user experience occurs

Error handling helps:

✅ Prevent crashes
✅ Show proper messages
✅ Make applications stable
✅ Handle unexpected situations safely



1. Syntax Error
Definition

Occurs when JavaScript code has invalid syntax.

Example
console.log("Hello"

Output:

SyntaxError
Why?

Closing bracket is missing.



2. Reference Error
Definition

Occurs when accessing variable that does not exist.

Example
console.log(name);

Output:

ReferenceError



3. Type Error
Definition

Occurs when operation is performed on wrong data type.

Example
const num = 10;

num.toUpperCase();

Output:

TypeError
Why?

toUpperCase() works only on strings.

try...catch in JavaScript
Definition

try...catch is used to handle errors without stopping the program.



Syntax


try {

}
catch(error) {

}



How It Works
Code inside try executes
If error occurs → control goes to catch
Program continues running

Basic Example


try {
  console.log(a);
}
catch(error) {
  console.log("Variable not found");
}



Output:

Variable not found
Explanation

a does not exist.

Instead of crashing:

👉 catch handles the error.

Accessing Error Object
try {
  console.log(a);
}
catch(error) {
  console.log(error);
}

Output:

ReferenceError: a is not defined
Error Object Properties
Property	Purpose
error.name	Error type
error.message	Error message


Example
try {
  console.log(a);
}
catch(error) {
  console.log(error.name);
  console.log(error.message);
}

Output:

ReferenceError
a is not defined

throw Keyword in JavaScript
Definition

throw is used to create custom errors manually.

Syntax
throw "Error Message";
Example
function checkAge(age) {

  if (age < 18) {
    throw "You are not eligible";
  }

  console.log("Eligible");
}

try {
  checkAge(15);
}
catch(error) {
  console.log(error);
}

Output:

You are not eligible
Explanation

We manually created an error using throw.

Better Example Using Error
function divide(a, b) {

  if (b === 0) {
    throw new Error("Cannot divide by zero");
  }

  return a / b;
}

try {
  console.log(divide(10, 0));
}
catch(error) {
  console.log(error.message);
}

Output:

Cannot divide by zero
Why Use Error Object?

Because it provides:

Error type
Message
Stack trace

Better for debugging.

------||------

Real-Life Example of Error Handling
function login(username) {

  try {

    if (!username) {
      throw new Error("Username is required");
    }

    console.log("Login Successful");

  }
  catch(error) {

    console.log(error.message);

  }
}

login("");

Output:

Username is required
Explanation

If user forgets username:

👉 Program handles error safely instead of crashing.
